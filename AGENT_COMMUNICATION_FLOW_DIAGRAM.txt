## 🔑 KEY DIFFERENCES (v1 vs v3)

### v1 — Sequential (Tuần Tự)
```
Orchestrator → Tool1 → Agent1 (wait)
            → Tool2 → Agent2 (wait)
            → Tool3 → Agent3 (wait)
            
Total: 245 + 380 + 520 = 1145ms
```

### v3 — Parallel (Song Song)
```
Orchestrator → [ThreadPoolExecutor]
     ├─→ Tool1 → Agent1 (start)
     ├─→ Tool2 → Agent2 (start)
     ├─→ Tool3 → Agent3 (start)
     └─→ Tool4 → Agent4 (start)
     
All run together → max(timings) = 520ms
→ 2x faster!
```

### v1 Flow Diagram:
```
Orchestrator.invoke()
    ↓
resp.tool_calls (list)
    ↓
Tool Router (for loop)
    ├─ tool_call["name"] = "search_agent"
    │  → AGENT_FN["search_agent"](args)
    │     → agent_search(query)
    │        → Tavily API / KB search
    │           → result
    │
    ├─ tool_call["name"] = "calculator_agent"
    │  → AGENT_FN["calculator_agent"](args)
    │     → agent_calculator(expr)
    │        → eval() with safe namespace
    │           → result
    │
    └─ [more agents...]
    ↓
ToolMessage[] added to state
    ↓
should_continue() check:
    if tool_calls AND iteration < MAX:
        → "tools" path → loop back
    else:
        → "end" path
    ↓
Orchestrator synthesize final answer
    ↓
Output to user
```
